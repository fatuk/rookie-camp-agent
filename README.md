# Rookie Camp Agent 🤖

Telegram-бот «Руби» на Gemini — напарница по вайбкодингу и геймдизайну для детей. Доступ по кодовому слову, дневной лимит сообщений на ученика.

## Как это работает

- Ребёнок пишет боту в Telegram кодовое слово → получает доступ.
- Дальше вайбкодит: бот помогает придумывать игры, механики и персонажей, даёт рабочий код и объясняет его.
- Сообщение «нарисуй пиксельного дракона» → бот генерирует картинку (концепт-арт, спрайты, локации).
- Можно прислать фото своего рисунка: с подписью — бот сразу нарисует по ней на основе арта; без подписи — запомнит рисунок и применит его к следующему «нарисуй …».
- У каждого ученика лимит сообщений в день (по умолчанию 50), чтобы не сжечь квоту Gemini.
- История диалога держится в памяти (после рестарта сервиса разговор начинается заново, доступ сохраняется в `users.json`).

## Подготовка

1. **Бот в Telegram**: напиши [@BotFather](https://t.me/BotFather) → `/newbot` → получи `BOT_TOKEN`.
2. **Ключ Gemini**: [aistudio.google.com/apikey](https://aistudio.google.com/apikey) → `GEMINI_API_KEY`.
3. Придумай `ACCESS_CODE` — кодовое слово для учеников.
4. (Необязательно) узнай свой Telegram id у [@userinfobot](https://t.me/userinfobot) → `ADMIN_ID` (даёт `/stats` и снимает лимит).

## Локальный запуск

```bash
npm install
cp .env.example .env   # заполни BOT_TOKEN, GEMINI_API_KEY, ACCESS_CODE
npm run dev
```

Без `WEBHOOK_URL` бот работает через polling — для разработки ничего настраивать не нужно. `npm run dev` сам подхватывает переменные из `.env`.

## Деплой на Render

1. Залей репозиторий на GitHub.
2. На [render.com](https://render.com): **New → Blueprint**, укажи репозиторий — Render прочитает `render.yaml`.
3. Заполни секреты `BOT_TOKEN`, `GEMINI_API_KEY`, `ACCESS_CODE` (и `ADMIN_ID`).
4. Готово. Render задаёт `RENDER_EXTERNAL_URL` автоматически — бот сам включит режим вебхука и зарегистрирует его в Telegram.

> **Free-план Render** усыпляет сервис после 15 минут тишины, но вебхук Telegram будит его входящим запросом — первое сообщение после паузы может отвечать ~30–60 секунд, дальше быстро.

> **Диск на Render эфемерный**: `users.json` (кто ввёл кодовое слово и счётчики) обнуляется при редеплое. Ученикам нужно будет один раз заново ввести кодовое слово. Если захочется постоянства — добавь Render Disk и задай `USERS_FILE=/data/users.json`.

## Команды бота

| Команда | Что делает |
|---|---|
| `/start` | Приветствие, запрос кодового слова |
| `/help` | Подсказка, как пользоваться |
| `/reset` | Начать разговор заново |
| `/limit` | Сколько сообщений осталось сегодня |
| `/stats` | (только админ) статистика по ученикам |

## Настройки (переменные окружения)

| Переменная | По умолчанию | Описание |
|---|---|---|
| `BOT_TOKEN` | — | токен от BotFather |
| `GEMINI_API_KEY` | — | ключ Gemini |
| `ACCESS_CODE` | — | кодовое слово для учеников |
| `GEMINI_MODEL` | `gemini-3.5-flash` | чат-модель Gemini |
| `GEMINI_IMAGE_MODEL` | `gemini-3.1-flash-image` | модель генерации картинок |
| `DAILY_LIMIT` | `50` | сообщений в день на ученика |
| `HISTORY_LENGTH` | `20` | реплик диалога в контексте модели |
| `ADMIN_ID` | — | Telegram id админа |
| `WEBHOOK_URL` | `RENDER_EXTERNAL_URL` | публичный URL (на Render не нужен) |
